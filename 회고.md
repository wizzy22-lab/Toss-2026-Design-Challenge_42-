# 회고 — 셸·일관성 패스

"완벽한 시간이 없어도 회의를 성립시킨다"는 논지는 그대로 두되, **개념을 절제하고
표면 간 모순을 없애는** 방향으로 정리했다. 뺀 것 위주로 기록한다 — 더한 것보다 뺀 게 많다.

## 뺀 것 (개념 절제)

- **회의 목적(결정/발산/공유) 분기 → 삭제.**
  목적으로 참석 방식을 게이트하던 구조가 과했다. 조율자가 회의를 만들 때마다
  "이건 무슨 성격이지"를 먼저 판단하게 만드는 건 부담이고, 실제로 목적이 바뀌면
  화상/비동기 허용이 통째로 뒤집히는 건 예측 불가능했다.
  → **참석 방식(화상·나중에 의견)은 항상 열려 있는 옵션**으로 바꿨다. 누가 어떤 방식으로
  참여할지는 목적이 아니라 **사람별 상황**에서 나온다. (`purpose`는 내부 기본값으로만
  남겨 계산 경로를 유지 — UI에는 노출하지 않음.)

- **후보 기간 드래그 → 삭제.**
  드래그 인터랙션은 정밀하지만 이 단계에 과했다. → 프리셋 칩(다음 주·이번 주·직접
  선택), 기본 '다음 주'. **직접 선택**을 골랐을 때만 시작/종료 탭 달력이 열린다.

- **소요시간 자유 입력 → 세그먼트 칩**(30분·1시간·1.5시간·2시간), 기본 1시간.
  기간 칩과 **같은 시각 언어**로 통일해 폼의 리듬을 맞췄다.

- **"내 차례에요" (차례 개념) → 삭제.**
  누가 먼저/다음이라는 순번은 없다. 참석자 데모 스위처는 순번이 아니라
  **"데모: 누구로 볼까요"** — 그냥 시점 전환일 뿐.

- **참석자 수동 추가 → 삭제.**
  채널에서 만든 회의니까 **채널 인원이 자동 초대**된다. 손으로 사람을 더하는 건
  이 흐름의 전제와 어긋난다. 대신 필요한 건 **외부 인원 초대**(다른 워크스페이스·외부,
  이메일·링크)라 그것만 남겼다.

## 바로잡은 모순 (신뢰도)

- **한소희: 외근 → 화상 자동 가정 → 삭제.**
  시스템이 "외근이면 화상 가능하겠지"라고 대신 정해주는 건 근거 없는 추정이었다.
  → 한소희가 **외근 시간마다 "화상 O / 화상 X"를 직접 표시**한다. 엔진도
  `remoteCapable`(본인 표시) **그리고** `allowRemote`(조율자 허용)가 모두 참일 때만
  화상으로 성립시킨다 — 자동 가정 경로를 제거했다.

- **미연동인데 외근이 캘린더에 뜨는 모순 → 정리.**
  미연동자(한소희)는 캘린더가 자동으로 채우지 않는다. 그의 제약(불가·외근·화상)은
  전부 **"본인이 직접 표시"**로 표현하고, 연동자의 **"일정 자동"**과 배지·문구로
  시각을 확실히 구분했다. 요청·응답·대시보드 세 표면에서 같은 규칙을 쓴다.

## 재배분한 부담

- **응답 상태는 공개, 응답 내용은 비공개.**
  누가 답했다/안 했다는 팀 전체에 보인다 — 독촉 부담을 한 사람이 아니라 모두가
  나눠 지도록. 반면 **무엇을 골랐는지**는 비공개 — 서로 눈치 보며 답을 맞추는
  게이밍을 막기 위해서다.

## 표면 일관성

- 좌측 DM 목록에 참석자 6명 전원 노출.
- 채널 잡담은 회의와 무관한 **일반 대화**로(진짜 메신저처럼) — 봇 카드만 회의 무대.
- 회의 만들기 CTA는 오른쪽 패널이 아니라 **컴포저가 위로 확장**되는 하단 시트
  (길면 모달처럼 스크롤) — "메신저가 1차 표면"을 인터랙션으로도 지켰다.

## 남긴 판단

- `purpose` 타입·엔진 경로는 **삭제하지 않고 내부 기본값('decision')으로 동결**했다.
  계산 로직을 건드리지 않아 회귀 위험이 없고, 나중에 목적 개념이 정말 필요해지면
  UI만 다시 붙이면 된다. 지금은 사용자에게 보이지 않는다.
- 필참은 **기본 전원**, 빼고 싶은 사람만 "선택으로" 내린다. "선택"(가중치 낮춤,
  참여는 함)과 "제외"(아예 빠짐)는 **다른 동작**으로 분리했다.

# 회고 — 추천 공개·인라인 확정·변경 재조율 패스

핵심은 두 가지 새 축이다: **관점(주최자↔참석자)**과 **변경(확정은 잠금이 아니다)**.
"메신저가 1차 표면"과 "부담 재배분"을 이 두 축으로 끝까지 밀었다.

## 추천을 에페메럴로 (애매함 제거)

- 예전엔 추천 카드가 채널에 그냥 떠서 "이게 공지인가 초안인가"가 애매했다.
  → **추천은 주최자에게만 보이는 에페메럴("나에게만 보여요")**. `confirmedKey===null`이면
  초안(주최자 전용), **정하기를 눌러야** 채널 공지로 승격된다. 구현상 별도 플래그 없이
  `confirmedKey`의 null 여부가 곧 "에페메럴 ↔ 공지" 경계다.
- 추천 개수는 **최선 하나 크게 + "다른 시간도 있어요 →"**. 답은 하나로 좁혀 주되
  선택권은 숨기지 않는다("→"는 대시보드 히트맵으로).

## 정하기 = 풀스크린 아님, 인라인

- 정하기가 풀스크린 영수증으로 튀지 않는다. 카드가 **그 자리에서 확정 상태**로 바뀌고
  채널에 공지 카드가 붙는다. 대시보드는 "자세히 보기"로 들어간 사람만 거기서 정한다.
  (풀스크린 `Confirm.tsx`는 해피패스에서 뺐다 — 삭제는 안 함.)

## 관점 = 데모의 두 번째 축

- 데모 스위처를 **관점(주최자/참석자) × 단계(만들기~변경)** 두 축으로 재편했다.
  참석자 관점엔 "받은 것"만 보인다 — 요청 카드(ⓐ)·확정 공지(ⓑ). 추천은 절대 새지
  않는다(주최자 전용). 사이드바 계정 페르소나도 관점 따라 바뀐다.

## 메시지 라이팅 (문구가 곧 가치)

- **요청**(응답 전): "…안 되는 시간만 톡 찍으면 끝나요(30초)" — 쉽고 빠르다는 안심,
  압박·죄책감 회피.
- **확정**: "다들 괜찮은 시간으로 맞췄어요. 한소희님은 화상으로 함께해요" — 누가
  양보했는지 콕 집지 않고, 화상 참여자를 2등으로 만들지 않는다.
- **리마인드**: "아직 2명이 시간을 안 알려줬어요" — 인원수만, **이름은 콕 집지 않음**
  (roster 칩으로 상태는 공개하되 프로즈 독촉은 익명). 공개 망신 회피.

## 변경/재조율 — "빠져나갈 구멍은 항상"

- 원칙: **확정은 잠금이 아니다.** 바꾸기 쉬워야 빠른 확정이 안심된다 → escape hatch가
  "속도" 가치를 떠받친다. 확정 카드엔 "참석 어려워요 / 시간 조정"이 **상시 노출**.
- 자유 텍스트 자동 감지는 **안 한다**(오작동 위험) — 채팅으로 "못 갈 것 같아요" 해도
  그 버튼으로 구조화. 받는 건 딱 둘: 누가 / 그 시간에 화상 되는지.
- **방해 최소 사다리**를 엔진(`reCoordinate`)이 계산하고 주최자가 고른다:
  ① 그 시간 그대로 그 사람만 화상(나머지 방해 0) → ② 필수 아니면 빼도 성립? →
  ③ 받아둔 응답으로 새 시간 재추천(재수집 없음). 검증: 정지훈(필수)이 화 15시 불가면
  ①(화상)과 ③(목 14시)이 뜨고 ②는 필수라 안 뜬다. 화상 불가면 ①이 사라진다.
- 전파: 정하면 모두 자동 알림 + 캘린더 갱신. **처음부터 다시가 아니다**(When2meet 대비).
  변경 공지는 **중립적**으로 — 특정인 탓이 안 나게.
- 슬롯 단위 "직접 불가·화상 가능"을 종일 외근과 구분하려고 `onsiteBlocked` 슬롯 개념을
  더했다(변경으로 정상 참석자가 그 슬롯만 화상이 되는 경우).

## 남긴 판단

- 변경 데모는 **정지훈(필수·대면)**이 화 15시에 참석 어려워지는 시나리오로 고정했다.
  필수라 ②가 막히고 ①③만 뜨는 게 사다리를 가장 잘 보여준다.
- `Confirm.tsx`(풀스크린 영수증)는 해피패스에서 뺐지만 코드는 남겨 뒀다 — 나중에
  "영수증 자세히"가 필요하면 다시 붙이면 된다.

# 회고 — 화상 제거·대면 전용·트레이드오프 투명 패스

가장 큰 절제. **참석 방식(화상·비동기) 개념을 통째로 들어냈다.** 회의 = 대면, 축은
'시간' 하나. 개념이 하나 줄면서 오히려 정직해졌다.

## 뺀 것 (개념 절제의 정점)

- **화상·비동기 참석 방식 전부 삭제.** `Mode`(onsite/remote/async/none) 타입,
  `ModePill`/`MODE_META`, 사람별 `allowRemote`/`allowAsync` 토글, `remoteCapable`/
  `onsiteBlocked`, `findActiveSuggestion`(한소희 화상 히어로) — 전부 제거. 한 사람은
  한 슬롯에 **되거나 안 되거나**(`isAvailable` 이진)다.
- **외근 = 그냥 '안 됨'.** "외근이지만 화상 가능" 옵션과 '화상 O/X' 토글을 없앴다.
  미연동자(한소희)가 직접 표시한 **불가**일 뿐, 화상 대안은 없다.
- `Confirm.tsx`(구 영수증)를 **삭제**했다(이미 해피패스에서 빠져 있었고 Mode에 의존).

## "완벽한 시간 없음"을 화상 없이 다루기

- 예전엔 완벽한 시간이 없으면 **화상으로 때웠다**(한소희 remote → 성립). 이제는
  **최선 + 트레이드오프를 정직하게** 보여준다: "완벽히 다 되는 시간은 없어요. 이 시간이
  최선 — 최민영님이 조금 불편(회피)." 조율자가 이해하고 정한다.
- **필수 전원 되는 시간이 아예 없으면** 감추지 않고 "필수 전원 되는 시간이 없어요"라고
  말하고 **레버**(후보 기간 넓히기 / 필수 인원 조정)를 준다. 에페메럴 추천·대시보드
  양쪽에서.
- **반응형의 재정의**: '화상 토글 → 재계산'이 아니라 **'참석자 입력이 바뀌면 →
  추천·트레이드오프가 실시간 갱신'**이 반응형 순간. `derived`가 `attendees` 변화에
  즉시 재계산하므로 이미 성립.

## 데이터 재튜닝 (핵심)

- 기존 busy 맵은 **화상 전제**라 화상을 빼면 성립 슬롯이 0개였다(모든 칸에 최소 1명
  불가 → 화상으로만 구제). 대면만으로:
  - 성립 3슬롯(수 14·월 11·금 15), **완벽(회피 0) 슬롯 0개** → 항상 트레이드오프.
  - 최선 = 수 14:00(회피 최민영 1명). `scratchpad/tune2.mjs`로 검증.
- 변경 사다리도 이 데이터로 검증: 정지훈(필수) 수 14 불가 → 재추천 월 11.

## 사다리 단순화

- 화상이 없으니 방해 최소 사다리에서 **①"화상으로 유지"가 사라졌다**. 남은 건:
  ① 선택 참석자면 그 시간 그대로 두고 빠짐(결과 공유) → ② 새 시간 재추천. 필수인데
  둘 다 안 되면 "후보 기간/필수 인원" 레버로.

## 남긴 판단

- `purpose`(회의 목적)를 **완전히 제거**했다(지난 라운드엔 내부 동결이었지만, 참석
  방식이 사라지면서 목적 게이트가 가릴 대상 자체가 없어짐 → 타입·상태·엔진에서 삭제).
- `Toggle` UI 컴포넌트는 남겨 뒀다 — 정족수·필참 등 다른 스위치에서 계속 쓴다.
