# MeetSync — 회의 성립 조율 프로토타입

"조율자 한 명이 동료 6명의 이질적 상황을 한눈에 이해하고, **완벽한 시간이 없어도** 회의를 성립시키게 돕는 도구."
When2meet(겹침만)·Clockwise(자동 결정)와 달리 — *가능한 시간*이 아니라 *모두가 괜찮은 최선*을 투명하게 고른다.

**회의는 대면 전용, 축은 오직 '시간' 하나.** 화상·비동기 같은 참석 방식 개념은 없다 — 한
사람은 한 슬롯에 **되거나(대면) 안 되거나** 둘 중 하나. "완벽한 시간 없음"은 화상으로
때우지 않고, **최선 + 트레이드오프(누가/왜 조금 불편)를 정직하게** 보여주는 것으로 다룬다.

## 메인 컨테이너 = 사내 메신저 셸

MeetSync는 독립 앱이 아니라 **사내 메신저 안의 앱/봇**이다. 모든 흐름이 채널(`#디자인팀`)
안에서 시작·발생한다 — "메신저가 1차 표면 · 결정을 앱에 가두지 않는다"를 물리적으로 구현.

- **셸(정적·뮤트)** = 좌측 사이드바(워크스페이스·채널·DM·MeetSync 앱) + 채널 헤더 + 메시지 영역 + 컴포저.
  슬랙의 *패턴*만 차용하고 브랜드는 복제하지 않음(무채색 차콜, 자체 인디고 액센트). 낮은 대비로 조용히.
- **MeetSync 카드 = 봇 메시지** = 밝은 무대. 실제 상호작용은 카드·오버레이·대시보드에서만.

## 실행

```bash
npm install
npm run dev      # http://localhost:5173
```

백엔드·로그인·DB 없음. 전부 mock 데이터 + 로컬 상태(React useReducer).

## 흐름 (전부 셸 안에서 시작)

1. **조율자** — 컴포저의 `＋ 회의 만들기` → 회의 만들기 시트(①) → 만들면 채널에 **요청 카드** 게시.
2. **참석자** — 채널의 요청 카드 → `응답하기` → 컴팩트 응답 오버레이(②, 불가·회피만 표시) → 제출 → 내 것만 보이는 영수증.
3. **전원 응답** → 추천이 **주최자에게만 보이는 에페메럴 카드**("나에게만 보여요")로 뜸: "완벽히 다 되는 시간은 없어요. 최선은 수 14:00 — 최민영님이 조금 불편(회피)."
4. **조율자** — 카드에서 원탭 `이 시간으로 정하기`(→ 인라인 확정 + 채널 공지) 또는 `다른 시간도 있어요 →`(대시보드).

상단 **데모 스위처**는 두 축 — **관점(주최자/참석자) × 단계(만들기·응답·추천·확정·변경)**.

## 셸 안의 표면들

- **회의 만들기 (①·하단 시트)** — 프리셋 기간 칩 + 소요시간 칩 + 채널 인원 자동 초대(필참 기본, 제외/선택). 목적·참석방식 개념 없음.
- **참석자 응답 (②·컴팩트 오버레이)** — 슬롯 탭 → 불가 → 회피(soft) → 해제. 외근일은 그냥 '안 됨'(화상 대안 없음). 제출 → 내 것만 보이는 영수증.
- **성립 대시보드 (Layer 2·전체 오버레이)** — 좌: 추천 Top3 / 중앙: 주간 히트맵(가능·회피·불가) / 우: 참석자 패널(필수↔선택)·정족수. **입력이 바뀌면 좌·중앙·트레이드오프가 즉시 재계산**(이게 반응형의 핵심).
- **확정 · 변경 (채널 인라인)** — 정하기=풀스크린 없이 카드가 그 자리에서 확정 + 채널 공지. 확정 카드엔 "참석 어려워요 / 시간 조정" 상시 노출(escape hatch).

## 핵심 개념 반영

- **제약 3층**: hard(불가·외근) / soft(회피) / 가중치(필수·선택) — 절대 뭉개지 않음.
- **대면 이진**: 한 슬롯에서 참석 가능/불가 둘 중 하나. "참석 방식"은 없다.
- **성립**: "필수 전원이 대면 가능 + 선택 N명 이상" → 성립. **완벽 = 성립 + 회피 0**(설계상 존재하지 않음 → 항상 트레이드오프).
- **트레이드오프 투명**: 완벽한 시간이 없다는 걸 감추지 않고, 최선인 이유(누가/왜 조금 불편)를 그대로 보여줌.
- **관점 분리**: 추천은 주최자 전용 에페메럴, 정해야 채널 공지. 참석자는 "받은 것"(요청·확정)만 봄.
- **변경 = 잠금 아님**: 확정 후에도 "참석 어려워요" → 방해 최소 사다리(선택자면 빼기 / 새 시간 재추천, 재수집 없음).
- **캘린더**: 연동자(5명)는 busy 자동, 미연동(한소희)은 외근·불가를 본인이 직접 표시.

## 검증된 시나리오

`scratchpad/tune2.mjs` + esbuild로 mock 맵을 검증:
- 대면만으로 **성립 슬롯 3개**(수 14:00·월 11:00·금 15:00), **완벽(회피 0) 슬롯은 0개** → "완벽한 시간 없음"이 기본, 최선 = 수 14:00(회피 최민영).
- 변경: **정지훈(필수)이 수 14:00 불가 → 재추천 = 월 11:00**(재수집 없이). 정지훈이 선택 참석자면 "빼기"도 성립.

## 로직 위치

- `src/shell/Shell.tsx` — 메신저 셸 + 데모 스위처(관점 × 단계). 페르소나 반영.
- `src/shell/Channel.tsx` — 관점별 타임라인. 주최자=요청진행/에페메럴 추천/확정공지/재조율, 참석자=요청 수신/확정 수신.
- `src/App.tsx` — 셸 + 채널 + 오버레이(만들기/응답/대시보드/변경진입) 조립.
- `src/engine.ts` — `isAvailable` / `evalSlot` / `topRecommendations` / `reCoordinate`. 추천·사다리 전부 상태에서 계산.
- `src/copy.ts` — 요청·확정·리마인드·트레이드오프·변경 문구.
- `src/data.ts` — 6명 mock, busy/외근/회피 시드(대면 전용, tune2로 검증).
- `src/store.tsx` — reducer + `useMemo` 재계산. `viewAs`(관점)·`confirmedKey`(에페메럴↔공지)·`change/lastChange`(변경).
